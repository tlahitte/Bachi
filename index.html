Qu’est-ce que le projet Bachi Taïko?

Bachi Taïko est une installation électronique et interactive mettant en relation une paire de Bachi avec un environnement visuel et auditif généré en temps réel. Par l’intermédiaire des Bachis et des capteurs électroniques implantés dans chacun d’eux, l’utilisateur renvoie des données à l’ordinateur qui sont traitées en temps réel pour faire venir évoluer l’environnement virtuel face à lui. Environnement virtuel en grande partie composé d’un rendu à partir d’une Xbox Kinect.

Installation

Des images du projet sont disponibles dans le fichier IMG associé

Matériel nécessaire :
— une paire de Bachi
— un Arduino Leonardo
— deux cellules photosensibles
— 4x résistance 100k
— 4x DEL (couleur similaire)
— un capteur de vibration Piezo
— un accéléromètre 3DOF (ADXL335) //uniquement l’accélération en Y est utilisé.
http://diopter.fr/index.php/technique/systeme-numerique/41-adx-335-et-arduino
— Kinect model n° 1414 //tous les modelés ne sont pas fonctionnel avec openKinect
http://shiffman.net/p5/kinect/

Logiciel et librairies nécessaires :
— Processing 2.2.1
[ ] serial
[ ] oscP5
[ ] openKinect
— Arduino 1.0.6
— Max/MSP 6.1.8

Montage électronique

Referez-vous aux images pour plus de précision sur le montage. Chacun des capteurs doit être branché sur une entrée analogique du Arduino pour la récupération de données.
Utilisez un diviseur de tension pour les deux cellules photosensibles. Uniquement l’axe des Y est utilisé sur l’accéléromètre.

Configuration du code

Chacun des patch (Arduino, P5 et max/MSP) est commentée.
@1 — Arduino exécute la lecture des capteurs et envoie leurs données via le port série.
@2 — Processing lit les données du port série, les calibrent, les utilisent pour altérer le visuel et renvoie une partie des données en OSC à Max/MSP
@3 — Max/MSP se sert des valeurs de vibration du Piezo et de l’accéléromètre pour altérer l’audio.

Bug fréquent : Allouez plus de 1024Mo de mémoire à Processing en allant dans son menu de préférences.

Lancement du code

[1] téléverser le code Arduino sur votre carte Leonardo.
[a] effectuer un premier test rapide en obstruant les cellules photosensibles, elles devraient allumer les DEL si elles ne perçoivent plus de lumières.
[2] Lancer le sketch Processing en mode présentation (shift + cmd + R)
Le sketch Processing contient une quantité d’options pour contrôler la bonne marche du logiciel.
[touche a] une fois relié à max/MSP, cette touche permet d’activer ou de couper l’audio.
[touche d] affichage un récapitulatif de débug du sketch (valeur reçue depuis l’Arduino, FrameRate actuel, FrameRate de la Kinect,...)
[touche c] permets de lancer l’outil de calibration des Bachis.
[touche UP et DOWN] règle le seuil de profondeur pris en compte par la Kinect.

Bug fréquent : changer la variable globale portName avec le nom de votre port série — visible depuis Arduino

[3] Lancer le patch max/MSP
[a] vérifié dans votre console max (cmd+M) que votre port UDP est bien fonctionnel.
[b] vérifier la bonne réception des valeurs des capteurs
